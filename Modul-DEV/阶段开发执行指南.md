## 阶段一：基础设施搭建（Phase 0）

### 目标
完成所有"零逻辑"的基础设施：依赖安装、目录结构、数据模型、核心工具模块、Agent 骨架、Prompt 模板。阶段结束时项目可正常启动，新表已创建。

### 任务清单

#### 1.1 依赖安装
```bash
pip install langgraph>=0.2.0 langgraph-checkpoint>=1.0.0 scikit-learn>=1.5.0 tenacity>=8.0
```
更新 `backend/requirements.txt`。

#### 1.2 目录结构创建
```bash
mkdir -p backend/app/agents
mkdir -p backend/app/prompts/multi_agent
```

**文件**：`backend/app/models/agent_state.py`
- `AgentRole` 枚举（9 种角色）

**文件**：`backend/app/models/multi_agent.py`
- `HumanReviewTask` 模型
- `CompilationCache` 模型
- `ABTestRecord` 模型
- `SemanticLink` 模型

**文件**：修改 `backend/app/models/compile_job.py`
- 追加字段：`compile_type`, `progress`, `current_agent`, `current_layer`, `final_score`, `compilation_log`

#### 1.4 核心工具模块（6 个文件）

**文件**：`backend/app/core/circuit_breaker.py`
- `CircuitState` 枚举 + `CircuitBreaker` 类（fail-fast + 半开恢复）

**文件**：`backend/app/core/llm_cache.py`
- `LLMCache` 类（内存 LRU + SQLite 持久化，懒初始化，路径统一 `./data/`）

#### 1.5 Prompt 模板（8 个 .md 文件）

`backend/app/prompts/multi_agent/` 目录下：

## 阶段二：后端核心引擎（Phase 1 MVP 后端）

### 目标
完成 LangGraph 图编排 + 全部 8 个 Agent 实现 + API 端点 + SSE 流。阶段结束时可通过 API 触发多智能体编译并获得完整 Wiki 输出。

### 任务清单

#### 2.1 LangGraph 图编排

**文件**：`backend/app/services/multi_agent_graph.py`
- `MultiAgentCompilationGraph` 类（`build()` 完整图 + `_decide_next()` 路由决策）
- 所有 8 个 LangGraph node wrapper 函数
- `_active_tracers` 全局注册表
- 注意：`_decide_next` 是**唯一**路由决策点，各 Agent 不再自行设置 `current_agent`

#### 2.2 Agent 函数完善

将阶段一的 8 个 Agent 骨架填充为完整实现：
- 所有 Agent 函数从 `prompts/multi_agent/` 加载 prompt 模板
- 统一使用 `llm_service.chat_completion(state["_model_config"], ...)` 调用
- 统一 JSON 解析保护（try/except + markdown code block 剥离 + fallback 默认值）
- 通过 `state["tracer"]` 记录 trace 事件（phase_start/llm_request/llm_response/phase_end）
- ResearcherPool 使用 `asyncio.gather` 并行执行 3 个研究视角

#### 2.3 API 端点

**文件**：`backend/app/api/v1/multi_agent_compile.py`
- `POST /api/v1/compile/multi-agent` — 触发多智能体编译
- `GET /api/v1/compile/jobs/{job_id}/stream` — SSE 流式进度 + trace
- `GET /api/v1/compile/jobs/{job_id}/trace` — 完整 trace JSON
- `run_multi_agent_compile()` 后台任务（完整流水线：Load memos → Build state → Run graph → Save results）

**修改** `backend/app/main.py`：
- 注册 `multi_agent_compile.router`

#### 2.4 Schema 追加

**修改** `backend/app/schemas/compile.py`：
- `MultiAgentCompileRequest` Pydantic model
- `HumanReviewRequest` Pydantic model

### 验收标准
- [ ] `POST /api/v1/compile/multi-agent` 返回 `job_id` 和 `status: "pending"`
- [ ] SSE stream 连接后能收到 progress 事件（含 `current_agent`, `current_layer`, `compilation_log`）
- [ ] 编译完成后 wiki_pages 表有新记录，memos 的 compiled 标记为 True
- [ ] 3 条 Memo → 至少 1 篇 Wiki 端到端成功

### 预估工作量
- 新建文件：2 个
- 修改文件：4 个（8 个 Agent 从骨架到完整）
- 核心代码行数：~1000 行

## 阶段四：迭代闭环 + 高级特性（Phase 3 + Phase 4）

### 目标
完成 Editor 修订回路、Linker 语义关联发现、Human-in-the-Loop 人工审核、增量编译、A/B 测试、质量评估。阶段结束时系统具备完整的多智能体闭环能力。

### 任务清单

#### 4.1 迭代修订闭环
- Editor → Reviewer×2 → Arbiter → (PASS → Linker | REVISE → Editor) 循环
- 修订次数上限控制（`max_revisions`）
- Arbiter 加权评分逻辑完善

#### 4.2 人工审核（HITL）

**文件**：`backend/app/services/human_review_manager.py`（完善阶段一骨架）
- `create_review_task()` — 创建审核任务 + SSE 通知前端
- `submit_decision()` — 处理批准/修订/终止
- `asyncio.Event` + 超时自动批准（5 分钟）
- `POST /api/v1/compile/human-review/{task_id}/submit` 端点

**完善** `HumanReviewPanel.vue`：
- 审核数据加载（评分、评审意见、Wiki 内容）
- 批准 → 继续 Linker 阶段
- 修订 → 用户编辑 Wiki 内容 → 重新进入写作/评审
- 终止 → 放弃本次编译

#### 4.3 Linker 语义关联
- 调用 `_get_existing_wikis_summary()` 获取已有 Wiki 摘要
- LLM 发现跨批次语义关联
- 高置信度（>=0.7）链接写入 `semantic_links` 表
- `get_graph_data()` 扩展：支持前端可视化语义链接

#### 4.4 增量编译（可选，按时间决定）

**文件**：`backend/app/services/incremental_compile_service.py`
- Memo 内容 hash → `compilation_cache` 表
- 未变更 Memo 跳过 LLM 调用，直接复用缓存

#### 4.5 质量评估

**文件**：`backend/app/services/evaluation_service.py`
- `WikiEvaluationService` 类
- 5 维评分：结构完整性 / 可读性 / 信息密度 / ROUGE-L / BERTScore
- 编译完成后自动评估（异步，不阻塞结果返回）

### 验收标准
- [ ] Editor → Arbiter 循环正常：低分自动修订，达标自动放行
- [ ] 超过 `max_revisions` 后强制终止修订
- [ ] 人工审核面板完整可用（批准/修订/终止 三条路径都通）

- [ ] 质量评估在编译完成后自动运行（日志可见评分）

## 关键风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| LangGraph 版本 API 变更 | 图定义语法不兼容 | 锁定 `langgraph>=0.2.0`，优先用手册中已验证的 API |
| scikit-learn 过大 | 安装慢、包体积大 | Coordinator 聚类为可选功能，embedding_model 未配置时跳过 |
| SSE 在前端断连 | 编译状态丢失 | `request.is_disconnected()` 检测 + 前端 EventSource `onerror` 重连 |
| LLM 返回非 JSON | Agent 解析失败 | 所有 Agent 统一 JSON 解析保护（code block 剥离 + try/except + fallback 默认值） |
| 多模型 API key 混乱 | LLM 调用失败 | `build_litellm_kwargs()` 已有按 provider 分派逻辑，新增 Agent 统一使用 |
