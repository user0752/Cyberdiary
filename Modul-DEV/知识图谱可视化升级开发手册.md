## 一、项目概述

### 1.1 现状与问题

多智能体编译引擎在编译完成后产出高质量的 Wiki 文章，但**缺少知识图谱可视化层**。当前存在的问题：

| 问题 | 表现 |
|------|------|
| **无图谱展示** | Researcher 抽取的 entities/relations 只在 LLM 内部流转，从未被可视化 |
| **可读性差** | 仅有 `[[双链]]` 文本标记，没有节点-边结构视图 |
| **交互性为零** | 纯静态页面，无拖拽/缩放/点击/搜索 |
| **视觉简陋** | 无设计系统、无主题、无动画 |

### 1.2 核心目标

构建一个 **2D/3D 可交互知识图谱**，将多智能体编译过程中抽取的知识结构直观呈现给用户。

| 维度 | 目标 |
|------|------|
| **数据** | 后端新增图谱聚合 API，将 entities + relations 组装为 nodes + edges |
| **可视化** | 力导向图谱，支持 2D (d3-force) 和 3D (Three.js) 双模式 |
| **交互** | 拖拽、缩放、悬停高亮、点击详情、搜索筛选 |
| **视觉** | 暗色/亮色双主题，玻璃拟态面板，流畅动画 |
| **实时** | 编译过程中图谱逐步"生长"的动画展示 |

### 1.3 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| **后端** | Python FastAPI（已有） | 复用现有项目后端 |
| **图谱数据聚合** | 新增 GraphDataService | 从 compilation_log + integrated_knowledge 聚合 |
| **2D 图** | D3-force + Canvas 自绘 | 灵活可控，无需额外依赖 |
| **3D 图** | Three.js（项目中已有） | 复用现有 Three.js 能力 |
| **前端框架** | Vue 3（已有） | 与项目现有一致 |
| **状态管理** | Pinia（已有） | 与项目现有一致 |
| **SSE** | EventSource（已有） | 复用现有编译流 |

## 二、数据接口层需求

### 2.1 图谱数据聚合服务

创建一个 `GraphDataService`，从已有的多智能体编译结果中提取并聚合图谱数据。

**输入数据来源**：

| 数据源 | 位置 | 包含内容 |
|--------|------|----------|
| `integrated_knowledge.entities` | CompilationState | 实体列表 (name, type, description) |
| `integrated_knowledge.relations` | CompilationState | 关系列表 (source, target, type, label) |
| `semantic_links` | 数据库 | 语义链接 (source_slug, target_slug, relation_type, confidence, reason) |
| `compilation_log` | CompilationState | Agent 执行日志（含 Researcher 各视角输出） |

**输出数据结构**：

```typescript
// 图谱数据根结构
interface KnowledgeGraph {
  nodes: GraphNode[];
  edges: GraphEdge[];
  meta: GraphMeta;
}

// 节点
interface GraphNode {
  id: string;              // 唯一标识，如 "entity_python"
  label: string;           // 显示名称，如 "Python"
  type: NodeType;          // 节点类型
  group: string;           // 所属分组（来自聚类）
  weight: number;          // 权重 0~1（影响节点大小）
  memoCount: number;       // 关联的 Memo 数量
  description: string;     // 简短描述
  sourceMemos: string[];   // 来源 Memo ID 列表
  originalEntity: any;     // 原始 entity 对象（保留扩展字段）
}

// 节点类型枚举
enum NodeType {
  TECHNOLOGY = "technology",
  CONCEPT = "concept",
  PERSON = "person",
  ORGANIZATION = "organization",
  TOOL = "tool",
  FRAMEWORK = "framework",
  LANGUAGE = "language",
  METHOD = "method",
  THEORY = "theory",
  OTHER = "other",
}

// 边
interface GraphEdge {
  id: string;              // 唯一标识
  source: string;          // 源节点 ID
  target: string;          // 目标节点 ID
  label: string;           // 关系标签，如 "属于"、"用于"
  type: EdgeType;          // 关系类型
  confidence: number;      // 置信度 0~1
  directed: boolean;       // 是否有方向
  sourceEdges: number;     // 该边的源节点出度
  targetEdges: number;     // 该边的目标节点入度
}

// 关系类型枚举
enum EdgeType {
  BELONGS_TO = "belongs_to",     // 属于
  USED_FOR = "used_for",         // 用于
  DEPENDS_ON = "depends_on",     // 依赖
  SIMILAR_TO = "similar_to",     // 相似
  RELATES_TO = "relates_to",     // 关联
  DERIVED_FROM = "derived_from", // 衍生自
  IMPLEMENTS = "implements",     // 实现
  REFERENCES = "references",     // 引用
}

// 图谱元数据
interface GraphMeta {
  totalNodes: number;
  totalEdges: number;
  nodeTypes: Record<NodeType, number>;  // 各类节点数量
  edgeTypes: Record<EdgeType, number>;  // 各类边数量
  maxConnectedComponent: number;        // 最大连通分量大小
  compilationId: string;                // 关联的编译任务 ID
  compiledAt: string;                   // 编译完成时间
}
```

### 2.2 新增 API 端点

| 端点 | 方法 | 功能 | 返回 |
|------|------|------|------|
| `GET /api/v1/compile/jobs/{job_id}/knowledge-graph` | GET | 获取完整图谱数据 | `KnowledgeGraph` |
| `GET /api/v1/compile/jobs/{job_id}/knowledge-graph/node/{node_id}` | GET | 获取单个节点详情 | `GraphNode + 关联 edges + 关联 Wiki 列表` |
| `GET /api/v1/compile/jobs/{job_id}/knowledge-graph/search?q={keyword}` | GET | 搜索节点 | `GraphNode[]`（模糊匹配 label） |
| `GET /api/v1/compile/jobs/{job_id}/knowledge-graph/filter?types=concept,technology` | GET | 按类型筛选节点+边 | `KnowledgeGraph`（过滤后的子图） |
| `GET /api/v1/compile/jobs/{job_id}/knowledge-graph/stream` | SSE | 图谱增量更新流 | 见下方 SSE 事件格式 |

**SSE 增量事件格式**：

```
// 阶段更新 —— 一批新节点/边到达
event: graph_update
data: {
  "event": "graph_update",
  "data": {
    "phase": "research" | "integrate" | "link" | "complete",
    "newNodes": GraphNode[],
    "newEdges": GraphEdge[],
    "removedNodes": string[],    // 去重后移除的节点 ID
    "removedEdges": string[],    // 去重后移除的边 ID
    "mergedNodes": {             // 去重后合并的节点
      "keep": string,            // 保留的节点 ID
      "removed": string[]        // 被合并掉的节点 ID
    }[]
  }
}

// 最终完成
event: graph_complete
data: {
  "event": "graph_complete",
  "data": KnowledgeGraph  // 完整最终图谱
}
```

### 2.3 数据聚合规则

**节点去重**：
- 相同 label 且相同 type 的 entity，合并为一个 node
- weight 取各 entity confidence 的加权平均值
- memoCount 累加
- sourceMemos 合并去重
- description 取较长者

**边去重**：
- 相同 source + target + type 的 relation，合并为一条边
- confidence 取最大值
- label 取较详细者

**节点分组**：
- 优先使用 Coordinator 的聚类分组结果
- 无聚类时按 entity.type 分组
- 分组名格式：`cluster_0`, `cluster_1` 或具体技术领域名

## 三、2D 力导向知识图谱

### 3.1 核心组件

创建一个 `KnowledgeGraph.vue` 组件，作为图谱的根容器。

**组件职责**：

```
KnowledgeGraph.vue
├── 图谱画布
│   ├── Canvas 渲染层（高性能，支持 500+ 节点）
│   ├── D3-force 力导向布局引擎
│   └── 节点/边的绘制逻辑
├── 工具栏
│   ├── 搜索框
│   ├── 类型筛选
│   ├── 关系类型筛选
│   ├── 布局重置
│   ├── 全屏切换
│   └── 2D/3D 模式切换
├── 图例
│   ├── 节点类型颜色映射
│   └── 边类型样式映射
└── 计数统计（总节点数 / 总边数）
```

### 3.2 布局引擎参数

使用 D3-force 模拟，以下为推荐的初始参数：

| 参数 | 值 | 作用 |
|------|----|------|
| `forceManyBody.strength` | -150 ~ -300 | 节点间斥力，越大越分散 |
| `forceManyBody.distanceMin` | 20 | 节点最小距离 |
| `forceManyBody.distanceMax` | 800 | 斥力作用范围上限 |
| `forceLink.distance` | 80 ~ 150 | 边长度（按边权重调节） |
| `forceLink.strength` | 0.3 ~ 0.8 | 边弹簧强度 |
| `forceCenter.x/y` | 画布中心 | 整体居中 |
| `forceCollide.radius` | node.weight * 5 + 15 | 节点碰撞半径 |
| `alphaDecay` | 0.02 | 布局冷却速度 |
| `velocityDecay` | 0.3 | 速度衰减系数 |

**布局过程**：
1. 初始随机分布
2. 前 50 帧快速收敛（high alpha）
3. 50~200 帧精细调整（alpha decay）
4. 200 帧后冻结（alpha < 0.001），用户可拖拽重新激活

### 3.3 节点渲染规范

**节点视觉**：

```
┌────────────────────────────────┐
│                                │
│         ● 节点圆形             │
│      ┌──────┐                  │
│      │ 标签  │   ← 文字标签     │
│      └──────┘                  │
│                                │
│  大小: 10px + weight * 20px    │
│  颜色: 按 type 映射            │
│  光晕: 悬停时显示 glow 效果    │
│                                  │
└────────────────────────────────┘
```

**节点大小公式**：
```
radius = BASE_RADIUS + weight * WEIGHT_SCALE
// BASE_RADIUS = 8, WEIGHT_SCALE = 25
// 范围: 8px ~ 33px
```

**复合节点（大权重节点）**：
- weight >= 0.7 的节点使用双重圆环效果
- 外环半透明 + 内环实心
- 外环半径 = radius * 1.3

### 3.4 边渲染规范

**边视觉**：


### 3.6 暗色/亮色双主题

通过 CSS 变量实现，图谱画布根据 `data-theme` 属性切换：

**暗色主题**（默认）：
```
画布背景: #0a0a1a (深空蓝黑)
节点文字: #f1f5f9 (冷白)
边:       半透明冷灰
glow:     各 type 对应高光色
```

**亮色主题**：
```
画布背景: #f8fafc (暖白)
节点文字: #0f172a (深灰)
边:       半透明深灰
glow:     各 type 对应高光色（透明度降低 50%）
画布网格: rgba(0,0,0,0.04) 浅灰网格线
```

### 3.7 画布功能

- Canvas 尺寸：自适应容器宽高，窗口变化时重绘
- DPI 适配：Canvas 按 `devicePixelRatio` 缩放，保证 Retina 屏幕清晰
- 节流渲染：requestAnimationFrame 驱动，布局计算和渲染分离
- 离屏渲染：静态帧缓存到离屏 Canvas，仅在布局变化时刷新
- 视口裁剪：只渲染视口可见区域内的节点和边（大数据量优化）

## 四、交互系统设计

### 4.1 基础交互

| 交互 | 触发 | 反馈 |
|------|------|------|
| **拖拽节点** | 鼠标按下节点并拖动 | 节点跟随鼠标，力导向动态重平衡 |
| **拖拽画布** | 鼠标按下空白区域并拖动 | 整个画布平移（pan） |
| **滚轮缩放** | 滚轮上下滚动 | 画布以鼠标位置为中心缩放，范围 0.3x ~ 5x |
| **双击空白** | 双击画布空白处 | 复位视角到初始位置 + 缩放 1x |
| **悬停节点** | 鼠标移动到节点上 | 节点放大 1.2x，显示 glow 光晕，邻接边和邻接节点高亮，非邻接节点透明度降至 0.15 |
| **悬停边** | 鼠标移动到边上 | 边加粗，显示关系标签 tooltip |
| **点击节点** | 鼠标点击节点 | 右侧滑出实体详情面板 |
| **点击空白** | 点击画布空白处 | 关闭详情面板 |

### 4.2 悬停高亮效果

悬停节点时，画布状态变化：

```
悬停前:                         悬停后:
  A ──→ B ──→ C                 A ──→ B ──→ C (高亮，不透明)
  │                              │
  v                              v
  D ──→ E                       D ──→ E
  (正常显示)                     (高亮，不透明)

  F ──→ G ──→ H                F ──→ G ──→ H (变暗至 0.15 透明度)
  │                              │
  v                              v
  I                              I (变暗至 0.15 透明度)
```

实现方式：
1. 获取悬停节点 + 其 1 跳邻接节点集合
2. 标记集合内节点为 `highlighted`
3. 渲染时 non-highlighted 节点/边透明度设为 0.15
4. highlighted 边的透明度提升到 0.9
5. 悬停节点自身放大 1.2x

### 4.3 工具栏

工具栏固定在图谱画布左上角或顶部：

```
┌─────────────────────────────────────────────────────────┐
│ 🔍 [搜索框...]  │  ├ 全选  │  ⊗ 清除  │  [布局重置] │
│                                                         │
│ 节点类型:  ● 技术(12)  ● 概念(8)  ● 人物(3)  ...      │
│ 关系类型:  □ 属于  □ 用于  □ 依赖  □ 相似  □ 关联      │
│                                                         │
│ 统计: 总节点 45  总边 78  最大簇 23                     │
└─────────────────────────────────────────────────────────┘
```

**搜索功能**：
- 输入即搜索（debounce 300ms），调 `/search` API
- 匹配节点高亮 + 脉冲动画 + 自动聚焦（缩放至节点位置）
- 搜索结果列表显示在搜索框下方下拉
- 支持正则或模糊匹配

**筛选功能**：
- 节点类型筛选：点击类型按钮切换显隐
- 关系类型筛选：勾选复选框切换边显隐
- 筛选后力导向自动重布局

**布局重置**：
- 复位力导向布局到初始状态
- 所有节点重新随机分布 + 重新模拟

### 4.4 全屏模式

- 点击全屏按钮，图谱容器进入 Fullscreen API
- 全屏时工具栏自动折叠为浮动按钮
- ESC 或再次点击退出全屏

### 4.5 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+F` | 聚焦搜索框 |
| `R` | 重置布局视角 |

### 4.6 移动端适配

- 触摸拖拽节点/画布（Touch Events）
- 双指缩放（Pinch Zoom）
- 长按节点显示详情（Long Press，替代 hover）

## 六、3D 沉浸模式

### 6.1 概述

作为 Premium 体验的高级模式，3D 图谱基于 Three.js 构建，提供沉浸式的知识探索体验。

### 6.2 3D 场景构成

**节点**：
- 使用 SphereGeometry 球体
- MeshStandardMaterial + Emissive 自发光材质
- 球体半径 = 2D radius * 0.4（3D 中缩小，避免遮挡）
- 自发光颜色同 type 颜色，强度 0.3
- 悬停时 emissiveIntensity 提升至 0.8

**边**：
- 使用 CatmullRomCurve3 曲线（带微小弧度，避免直线生硬）
- LineBasicMaterial 半透明材质
- 箭头使用 ConeGeometry 锥体指向 target
- 高置信边附带流动粒子效果（小粒子沿路径运动）

**背景**：
- 星空粒子系统（BufferGeometry + Points）
- 粒子数量 1000~2000
- 随机分布在半径 500 的球壳内
- 粒子颜色 #ffffff，大小 0.5~2px

**光照**：
- AmbientLight 环境光 (0x404060, 0.5)
- DirectionalLight 方向光 (0xffffff, 1.0, 位置 (10, 20, 10))
- HemisphereLight 半球光 (sky: 0x404080, ground: 0x202020, 0.4)

**后期**：
- UnrealBloomPass 泛光效果
  - strength: 0.3
  - radius: 0.5
  - threshold: 0.1
- 选中/悬停节点 bloom 动态提升

### 6.3 3D 交互

| 交互 | 操作 | 反馈 |
|------|------|------|
| **旋转** | 鼠标拖拽（OrbitControls） | 场景绕 Y 轴旋转 |
| **缩放** | 滚轮 | 镜头拉近拉远 |
| **平移** | 右键拖拽 | 场景平移 |
| **聚焦节点** | 单击节点 | 镜头平滑移动到节点前方 + 放大 |
| **显示标签** | 悬停节点 | CSS2DRenderer 显示标签浮层 |
| **自动旋转** | 点击"自动旋转"按钮 | 场景以 Y 轴缓慢自转 |
| **重置视角** | 双击空白 | 回到初始视角 |

### 6.4 2D ↔ 3D 模式切换

```
┌─ Mode Switcher ─────────────────────┐
│  [◈ 2D 图谱]  [◆ 3D 沉浸]          │
└────────────────────────────────────┘
```

- 切换时图谱数据不变，仅切换渲染引擎
- 2D → 3D：节点从平面升起，散开到 3D 球面空间，动画 1s
- 3D → 2D：节点从 3D 降落到平面，动画 1s
- 保持当前选中/悬停状态跨模式同步

### 6.5 3D 性能要求

- 目标帧率：60fps（GPU 加速）
- 节点上限：300 节点 + 500 边
- 使用 InstancedMesh 优化同类型节点渲染
- 使用 FrustumCulling 剔除视口外物体
- LOD（Level of Detail）：远距离节点简化渲染
- 超过上限时降级为 2D 模式

## 八、实时图谱生长动画

### 8.1 概念

在编译过程中，图谱不是一次性出现的，而是随着 Agent 的推进**逐步生长**的：

```
时间轴 →
[空画布] → [Researcher1 完成 → 5 个节点出现] → [Researcher2 完成 → 3 个节点+4 条边]
→ [Integrator 完成 → 去重合并, 节点重新排列] → [Linker 完成 → 新增语义链接边]
→ [编译完成 → 最终图谱 + 布局稳定]
```

### 8.2 实现方式

1. 前端连接已有的 SSE `/stream` 端点
2. 从 `trace` 事件中提取 Agent 阶段变化
3. 每次 `graph_update` SSE 事件到达时，增量更新图谱数据
4. 新节点以动画方式"弹入"（scale 0→1 + opacity）
5. 新边以"绘制"动画（stroke-dashoffset → 0）
6. 去重合并时，节点"消失"（scale→0）或"移动"（位置过渡）

### 8.3 生长动画类型

| 事件 | 动画 | 时长 |
|------|------|------|
| 新节点出现 | scale 0→1 + rotate 360° (轻量) | 600ms |
| 新边绘制 | stroke-dashoffset 递减 | 500ms |
| 节点移动 | position 插值过渡 | 800ms |
| 节点合并 | 旧节点 fade out → 新节点 pop in | 400ms |
| 节点消失 | scale 1→0 + opacity 1→0 | 300ms |
| 阶段切换指示 | 画布顶部进度条更新 + 标签变化 | 即时 |

### 8.4 阶段指示器

在图谱画布顶部显示当前编译阶段进度条：

```
┌─ Compilation Progress ──────────────────────────────┐
│  🔍 研究阶段    🧩 整合阶段    ✍️ 写作阶段    🔗 链接阶段  │
│  ████████████████░░░░░░░░░░░░░░░░░░░░  45%           │
│  ── 当前: 研究者 #2 分析中...                        │
└──────────────────────────────────────────────────────┘
```

- 已完成阶段显示绿色高亮
- 当前阶段显示活跃动画（闪烁或脉冲）
- 待完成阶段显示灰色
- 鼠标悬停显示该阶段详情

### B. 关键实现注意事项

1. **Canvas 渲染优先于 DOM**：图谱画布必须使用 Canvas（或 WebGL）渲染，不能使用 DOM 元素绘制大量节点，DOM 在 100+ 节点时性能急剧下降
2. **RAF 驱动渲染循环**：所有渲染操作由 `requestAnimationFrame` 驱动，不与布局引擎同步阻塞
3. **D3-force 和渲染分离**：布局模拟在 worker 或离屏运行，渲染线程只读取位置数据
4. **大数据量降级**：节点数 > 500 时自动隐藏标签、降低帧率到 30fps、关闭 glow 效果
5. **内存管理**：Three.js 场景切换时手动 dispose geometry/material/texture，防止 GPU 内存泄漏
6. **错误处理**：API 请求失败时显示友好提示，3D WebGL 不支持时自动降级到 2D
7. **无障碍**：节点标注 aria-label，键盘可操作，提供图谱的文本替代描述

### C. 与现有系统的集成点

| 集成点 | 位置 | 方式 |
|--------|------|------|
| 编译完成后触发图谱加载 | `CompileResult.vue` → `KnowledgeGraph.vue` | 通过 `job_id` prop 传递 |
| 图谱节点→Wiki 跳转 | `EntityDetailPanel` 中的关联 Wiki 链接 | 调用现有 Wiki 路由 |
| 图谱节点→Memo 追溯 | `EntityDetailPanel` 中的来源 Memo | 调用现有 Memo API |
| 编译 SSE 流 | `MultiAgentCompileView.vue` → `KnowledgeGraph.vue` | 通过 store 共享 SSE 数据 |
| 主题同步 | 全局 `data-theme` → 图谱画布 | CSS 变量自动响应 |
