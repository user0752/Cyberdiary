## 1. 项目愿景

### 1.1 核心问题

传统知识管理工具（Notion、Obsidian、Logseq）存在共同问题：

- **被动消费**：阅读是单向的，缺乏主动回忆
- **枯燥乏味**：长时间阅读导致注意力下降，遗忘率高
- **缺乏反馈**：无法量化"掌握程度"，不知道哪里是知识盲区

### 1.2 解决方案

将 Wiki 编译后的结构化知识，转化为**可玩的知识塔防游戏**：

```
Wiki 知识图谱  ──→  游戏地图
知识概念      ──→  防御塔
知识误区      ──→  敌人
主动答题      ──→  防御攻击
遗忘曲线      ──→  敌人生成频率
```

### 1.3 技术亮点（求职核心卖点）

| 技术 | 应用场景 | 求职标签 |
|------|---------|---------|
| **LangGraph Multi-Agent** | 5 个 Agent 协作运行游戏 | Multi-Agent System |
| **RAG（检索增强生成）** | Knowledge Agent 检索 Wiki | RAG / Vector Search |
| **LLM Function Calling** | Agent 调用工具（查 DB、搜 Wiki） | Tool Use / Agent |
| **LangChain** | Agent 编排、Prompt 模板管理 | LangChain |
| **ReAct / Plan-and-Execute** | Game Master 决策模式 | Agent 推理框架 |
| **Phaser.js + Vue3** | 前端游戏引擎集成 | 全栈 / 游戏开发 |
| **FastAPI + SQLAlchemy** | 后端异步 API | Python 后端 |
| **LiteLLM** | 统一 LLM 路由（DeepSeek/OpenAI/Ollama） | LLM 工程化 |

#### Agent 1：Game Master Agent（游戏主持人）

**职责：**
- 监控游戏全局状态
- 根据玩家表现动态调整难度
- 编排其他 Agent 的调用顺序
- 决定下一波敌人的类型和强度

**状态定义：**
```python
from typing import TypedDict, List, Optional
from enum import Enum

class Difficulty(str, Enum):
    EASY = "easy"
    MEDIUM = "medium"
    HARD = "hard"
    BOSS = "boss"

class WaveConfig(TypedDict):
    wave_number: int
    difficulty: Difficulty
    enemy_types: List[str]       # ["forgetter", "confuser"]
    focus_concepts: List[int]    # Wiki 节点 ID
    boss_enabled: bool

class GameState(TypedDict):
    current_wave: int
    player_health: int
    player_score: int
    recent_accuracy: float        # 最近 10 题正确率
    weak_concepts: List[int]     # 常错概念 ID
    knowledge_mastery: dict       # {concept_id: mastery_score}
    game_events: List[str]       # 事件日志
    next_wave_config: Optional[WaveConfig]
```

**核心决策逻辑：**
```python
# backend/app/agents/game_master.py

from langchain.agents import AgentExecutor, create_react_agent
from langchain_core.prompts import ChatPromptTemplate
from langchain.tools import Tool
from langgraph.graph import StateGraph, END

class GameMasterAgent:
    """
    Game Master：游戏主持人 Agent
    使用 ReAct 模式进行决策
    """
    
    def __init__(self, llm):
        self.llm = llm
        self.tools = self._init_tools()
        self.agent_executor = self._init_agent()
    
    def _init_tools(self) -> List[Tool]:
        return [
            Tool(
                name="get_player_profile",
                func=self.get_player_profile,
                description="获取玩家当前状态：正确率、知识掌握度、最近表现"
            ),
            Tool(
                name="get_weak_concepts",
                func=self.get_weak_concepts,
                description="获取玩家知识盲区列表"
            ),
            Tool(
                name="calculate_difficulty",
                func=self.calculate_difficulty,
                description="根据玩家状态计算下一波难度"
            ),
        ]
    
    def decide_next_wave(self, state: GameState) -> GameState:
        """
        核心决策：下一波该怎么设计？
        返回更新后的 GameState
        """
        prompt = ChatPromptTemplate.from_messages([
            ("system", """你是一个塔防游戏的主持人。
根据玩家当前状态，决定下一波敌人的配置。

决策规则：
1. 正确率 < 40% → 降低难度，出 Forgetter 类敌人
2. 正确率 40-70% → 正常难度，混合敌人类型
3. 正确率 > 70% → 提高难度，可以出 Boss
4. 有知识盲区 → 针对性出该概念的敌人
5. 连续 3 波高正确率 → 触发 Boss 战

输出 JSON 格式：
{{
    "difficulty": "easy|medium|hard|boss",
    "enemy_types": ["forgetter", "confuser"],
    "focus_concepts": [1, 2, 3],
    "boss_enabled": true/false,
    "reason": "决策理由"
}}"""),
            ("human", "玩家状态：{state}")
        ])
        
        # 调用 LLM 决策
        response = self.llm.invoke(prompt.format(state=state))
        config = self._parse_response(response)
        
        # 更新状态
        state["next_wave_config"] = config
        state["current_wave"] += 1
        
        return state
    
    def calculate_difficulty(self, state: GameState) -> Difficulty:
        """基于规则的难度计算（备选方案，不依赖 LLM）"""
        accuracy = state["recent_accuracy"]
        wave = state["current_wave"]
        
        if accuracy < 0.4:
            return Difficulty.EASY
        elif accuracy > 0.8 and wave % 5 == 0:
            return Difficulty.BOSS
        elif accuracy > 0.6:
            return Difficulty.HARD
        else:
            return Difficulty.MEDIUM
```

**LangGraph 工作流集成：**
```python
# backend/app/agents/game_workflow.py

from langgraph.graph import StateGraph, END

def build_game_workflow():
    """构建游戏 Agent 协作图"""
    workflow = StateGraph(GameState)
    
    # 添加 Agent 节点
    workflow.add_node("game_master", GameMasterAgent().decide_next_wave)
    workflow.add_node("spawn_enemies", EnemySpawner().spawn)
    workflow.add_node("wait_for_answers", GameLoop().wait_for_answers)
    workflow.add_node("check_wave_clear", GameLoop().check_wave_clear)
    
    # 定义流转逻辑
    workflow.set_entry_point("game_master")
    workflow.add_edge("game_master", "spawn_enemies")
    workflow.add_edge("spawn_enemies", "wait_for_answers")
    workflow.add_conditional_edges(
        "wait_for_answers",
        lambda s: "check_wave_clear" if s["all_enemies_defeated"] else "wait_for_answers"
    )
    workflow.add_conditional_edges(
        "check_wave_clear",
        lambda s: END if s["game_over"] else "game_master"
    )
    
    return workflow.compile()
```

**职责：**

```python
# backend/app/agents/knowledge_agent.py

from langchain.tools import BaseTool
from sqlalchemy.ext.asyncio import AsyncSession

class KnowledgeAgent:
    """
    Knowledge Agent：知识检索官
    负责将 Wiki 知识图谱转化为游戏元素
    """
    
    def __init__(self, db: AsyncSession):
        self.db = db
    
    class SearchWikiTool(BaseTool):
        name = "search_wiki"
        description = "语义搜索 Wiki 节点，输入查询文本，返回相关概念列表"
        
        def _run(self, query: str) -> list:
            # 使用向量搜索（如果已配置）或全文搜索
            return WikiService.semantic_search(self.db, query, top_k=5)
    
    class GetRelatedConceptsTool(BaseTool):
        name = "get_related_concepts"
        description = "获取指定概念的所有关联概念（来自 Wiki 双链）"
        
        def _run(self, concept_id: int) -> list:
            return WikiService.get_related(self.db, concept_id)
    
    class BuildEnemyProfileTool(BaseTool):
        name = "build_enemy_profile"
        description = "将 Wiki 节点转化为敌人配置"
        
        def _run(self, concept_id: int) -> dict:
            node = WikiService.get_node(self.db, concept_id)
            related = WikiService.get_related(self.db, concept_id)
            
            # 混淆怪：从关联概念中选一个作为干扰项
            confuse_with = random.choice(related) if related else None
            
            return {
                "name": node.title,
                "description": node.summary,
                "wiki_node_id": concept_id,
                "enemy_type": self._infer_enemy_type(node),
                "difficulty": node.complexity_score or 3,
                "confuse_with": confuse_with,
                "hp": 50 + (node.complexity_score or 3) * 20,
                "reward_xp": 10 * (node.complexity_score or 3),
            }
        
        def _infer_enemy_type(self, node) -> str:
            """根据节点类型推断敌人类型"""
            if node.node_type == "definition":
                return "forgetter"
            elif node.node_type == "comparison":
                return "confuser"
            elif node.node_type == "causal":
                return "blinder"
            else:
                return "forgetter"
```

**RAG 检索实现（进阶）：**
```python
# backend/app/services/rag_service.py

from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

class RAGService:
    """
    RAG 服务：将 Wiki 内容向量化，支持语义检索
    MVP 阶段使用本地嵌入模型，无需外部 API
    """
    
    def __init__(self):
        # 使用本地多语言模型（支持中文）
        self.model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')
        self.embeddings_cache = {}  # {wiki_node_id: embedding}
    
    async def build_index(self, db: AsyncSession):
        """为所有 Wiki 节点建立向量索引"""
        nodes = await WikiService.get_all_nodes(db)
        for node in nodes:
            text = f"{node.title}\n{node.content}"
            embedding = self.model.encode(text)
            self.embeddings_cache[node.id] = embedding
    
    async def semantic_search(self, db: AsyncSession, query: str, top_k: int = 5) -> List[int]:
        """语义搜索：返回最相关的 Wiki 节点 ID"""
        query_embedding = self.model.encode(query)
        
        # 计算相似度
        similarities = []
        for node_id, node_embedding in self.embeddings_cache.items():
            sim = cosine_similarity([query_embedding], [node_embedding])[0][0]
            similarities.append((node_id, sim))
        
        # 排序并返回 top_k
        similarities.sort(key=lambda x: x[1], reverse=True)
        return [node_id for node_id, _ in similarities[:top_k]]
```

#### Agent 3：Question Agent（出题官）

**职责：**
- 根据玩家画像和知识节点，生成个性化题目
- 支持多种题型（选择题、判断题、填空题、场景题）
- 题目难度自适应

**题目生成 Prompt 工程：**
```python
# backend/app/agents/question_agent.py

from langchain_core.output_parsers import PydanticOutputParser
from pydantic import BaseModel, Field
from typing import Optional, List

class Question(BaseModel):
    """题目结构化输出"""
    question_type: str = Field(description="'mcq'|'tf'|'fill'|'scenario'")
    question_text: str = Field(description="题目正文")
    options: Optional[List[str]] = Field(description="选项（MCQ 时必填）")
    correct_answer: str = Field(description="正确答案")
    explanation: str = Field(description="详细解释（答错后展示）")
    wiki_node_id: int = Field(description="关联的 Wiki 节点 ID")
    difficulty: int = Field(description="难度 1-5")

class QuestionAgent:
    """
    Question Agent：出题官
    根据玩家状态和知识节点，生成个性化题目
    """
    
    def __init__(self, llm):
        self.llm = llm
        self.parser = PydanticOutputParser(pydantic_object=Question)
    
    def generate_question(
        self, 
        wiki_node: dict, 
        player_profile: dict,
        question_type: str = "mcq"
    ) -> Question:
        """
        生成一道题目
        
        Args:
            wiki_node: Wiki 节点（标题、内容、类型）
            player_profile: 玩家画像（历史正确率、常错类型、掌握度）
            question_type: 题目类型
        """
        
        prompt = ChatPromptTemplate.from_messages([
            ("system", """你是一个专业的出题专家，擅长根据知识点生成高质量题目。

出题原则：
1. 题目必须基于提供的 Wiki 内容，不能超纲
2. 干扰项要有迷惑性（基于真实常见误区）
3. 题目难度匹配玩家的知识掌握度
4. 输出严格的 JSON 格式

{format_instructions}"""),
            ("human", """
请为以下知识点生成一道 {question_type} 题目：

【知识点】
标题：{title}
内容：{content}
知识类型：{node_type}

【玩家画像】
历史正确率：{accuracy}
常错类型：{error_pattern}
该知识点掌握度：{mastery}

要求：
- 难度适配玩家水平（掌握度低 → 基础题；掌握度高 → 应用题）
- 如果是混淆类知识点，重点考察区分能力
""")
        ])
        
        chain = prompt | self.llm | self.parser
        
        result = chain.invoke({
            "question_type": question_type,
            "title": wiki_node["title"],
            "content": wiki_node["content"][:1000],  # 截断，避免超 token
            "node_type": wiki_node["node_type"],
            "accuracy": player_profile["accuracy"],
            "error_pattern": player_profile.get("error_pattern", "无"),
            "mastery": player_profile.get("mastery", 0.5),
            "format_instructions": self.parser.get_format_instructions(),
        })
        
        return result
    
    def batch_generate(self, wiki_nodes: List[dict], count: int = 5) -> List[Question]:
        """批量生成题目（用于预生成，减少游戏时延迟）"""
        questions = []
        for node in wiki_nodes[:count]:
            try:
                q = self.generate_question(node, {"accuracy": 0.5})
                questions.append(q)
            except Exception as e:
                print(f"生成题目失败（节点 {node['id']}）：{e}")
        return questions
```

#### Agent 4：Tutor Agent（Socratic 辅导官）

**职责：**
- 玩家答错时，不直接给答案，而是引导思考
- 使用 Socratic 教学法（提问 → 提示 → 展示原文）
- 根据错误类型采取不同辅导策略

**Socratic 辅导流程：**
```python
# backend/app/agents/tutor_agent.py

from enum import Enum

class ErrorType(str, Enum):
    CONFUSION = "confusion"      # 概念混淆
    PARTIAL = "partial"          # 部分理解
    COMPLETE = "complete"        # 完全不懂
    CARELESS = "careless"        # 粗心错误

class TutorAgent:
    """
    Tutor Agent：Socratic 辅导官
    答错时不直接给答案，而是引导玩家思考
    """
    
    def diagnose_error(
        self, 
        question: Question, 
        user_answer: str, 
        correct_answer: str
    ) -> ErrorType:
        """诊断错误类型"""
        # 简单启发式规则（可升级为 LLM 判断）
        if self._is_confusion(question, user_answer):
            return ErrorType.CONFUSION
        elif self._is_partial(user_answer, correct_answer):
            return ErrorType.PARTIAL
        else:
            return ErrorType.COMPLETE
    
    def handle_wrong_answer(
        self, 
        question: Question,
        user_answer: str,
        wiki_node: dict
    ) -> dict:
        """
        处理错误答案，返回辅导内容
        
        Returns:
            {
                "type": "hint|comparison|excerpt",
                "content": "...",
                "next_action": "retry|show_answer|unlock_tower"
            }
        """
        error_type = self.diagnose_error(question, user_answer, question.correct_answer)
        
        if error_type == ErrorType.CONFUSION:
            return {
                "type": "comparison",
                "content": self._generate_comparison(question),
                "next_action": "retry"
            }
        
        elif error_type == ErrorType.PARTIAL:
            hint = self._generate_hint(question, user_answer)
            return {
                "type": "hint",
                "content": hint,
                "next_action": "retry"
            }
        
        else:
            excerpt = self._get_wiki_excerpt(wiki_node)
            return {
                "type": "excerpt",
                "content": excerpt,
                "next_action": "show_answer"
            }
    
    def _generate_hint(self, question: Question, user_answer: str) -> str:
        """生成提示（不直接给答案）"""
        prompt = f"""
玩家答错了这道题：
题目：{question.question_text}
玩家答案：{user_answer}
正确答案：{question.correct_answer}

请给出一个提示（hint），帮助玩家思考，但**不要直接给出答案**。
提示应该：
1. 引导玩家回忆相关概念
2. 指出思考方向
3. 不超过 50 字
"""
        return self.llm.invoke(prompt)
    
    def _generate_comparison(self, question: Question) -> str:
        """生成对比表（用于混淆类错误）"""
        pass
```

#### Agent 5：Analytics Agent（学习分析师）

**职责：**
- 每局游戏结束后，分析玩家表现
- 发现知识盲区模式
- 生成个性化学习路径推荐

**分析逻辑：**
```python
# backend/app/agents/analytics_agent.py

class AnalyticsAgent:
    """
    Analytics Agent：学习分析师
    后台运行，不阻塞游戏流程
    """
    
    async def post_game_analysis(
        self, 
        db: AsyncSession,
        game_session_id: int
    ) -> dict:
        """
        局后分析：生成学习报告
        """
        # 1. 获取本局游戏数据
        session = await GameService.get_session(db, game_session_id)
        answers = await GameService.get_session_answers(db, game_session_id)
        
        # 2. 计算知识掌握度
        mastery_matrix = self._compute_mastery(answers)
        
        # 3. 发现知识依赖链中的薄弱点
        weak_chains = self._find_dependency_gaps(db, mastery_matrix)
        
        # 4. 生成推荐
        recommendations = self._generate_recommendations(
            mastery_matrix, weak_chains
        )
        
        # 5. 保存分析结果
        await AnalyticsService.save_analysis(
            db, game_session_id, 
            mastery_matrix, weak_chains, recommendations
        )
        
        return {
            "mastery_matrix": mastery_matrix,
            "weak_chains": weak_chains,
            "recommendations": recommendations
        }
    
    def _compute_mastery(self, answers: List[dict]) -> dict:
        """计算知识掌握度（基于 Item Response Theory 简化版）"""
        mastery = {}
        for answer in answers:
            concept_id = answer["wiki_node_id"]
            if concept_id not in mastery:
                mastery[concept_id] = {
                    "correct_count": 0,
                    "total_count": 0,
                    "avg_response_time": 0,
                    "last_answered": None
                }
            
            m = mastery[concept_id]
            m["total_count"] += 1
            if answer["is_correct"]:
                m["correct_count"] += 1
            m["avg_response_time"] = (
                m["avg_response_time"] * (m["total_count"] - 1) 
                + answer["response_time"]
            ) / m["total_count"]
            m["last_answered"] = answer["created_at"]
        
        for concept_id, m in mastery.items():
            accuracy = m["correct_count"] / m["total_count"]
            speed_bonus = max(0, 1 - m["avg_response_time"] / 30)
            mastery[concept_id] = accuracy * 0.7 + speed_bonus * 0.3
        
        return mastery
    
    def _find_dependency_gaps(
        self, 
        db: AsyncSession, 
        mastery_matrix: dict
    ) -> List[dict]:
        """发现知识依赖链中的薄弱点"""
        dependencies = await WikiService.get_dependencies(db)
        
        weak_chains = []
        for dep in dependencies:
            prereq_id = dep["prereq_id"]
            target_id = dep["target_id"]
            
            prereq_mastery = mastery_matrix.get(prereq_id, 0)
            target_mastery = mastery_matrix.get(target_id, 0)
            
            if prereq_mastery < 0.6 and target_mastery < 0.7:
                weak_chains.append({
                    "prereq_id": prereq_id,
                    "target_id": target_id,
                    "gap_score": 0.6 - prereq_mastery,
                    "recommendation": f"建议先巩固「{dep['prereq_title']}」"
                })
        
        return weak_chains
```

    new_state = await GameMasterAgent().decide_next_wave(db, state)

    session_id: int,
    question_id: int,
    user_answer: str,
    response_time: float,
    db: AsyncSession = Depends(get_db)
):

    question = await GameService.get_question(db, question_id)
    is_correct = (user_answer.strip() == question.correct_answer.strip())
    
    await GameService.record_answer(
        db, session_id, question_id, user_answer, is_correct, response_time
    )
    
    if not is_correct:
        wiki_node = await WikiService.get_node(db, question.wiki_node_id)
        tutor_response = await TutorAgent().handle_wrong_answer(
            question, user_answer, wiki_node
        )
        return {
            "is_correct": False,
            "tutor_response": tutor_response
        }
    
    damage = await GameService.calculate_damage(db, question_id)
    return {
        "is_correct": True,
        "damage": damage,
        "explanation": question.explanation
    }

### 9.1 简历描述模板

```
CyberDefender — 知识塔防游戏化学习平台（独立开发）
技术栈：LangGraph · LangChain · RAG · FastAPI · Vue3 · Phaser.js

• 基于 LangGraph 构建了 5 个协作 Agent（Game Master / Knowledge / 
  Question / Tutor / Analytics），实现自适应游戏难度和个性化辅导
• 使用 RAG（检索增强生成）技术，将个人 Wiki 知识库转化为游戏素材，
  实现知识图谱 → 游戏地图的自动生成
• 设计了 Socratic 辅导 Agent，玩家答错时引导思考而非直接给答案，
  结合艾宾浩斯遗忘曲线实现间隔重复
• 全栈实现：FastAPI 异步后端 + Vue3/Phaser.js 前端游戏引擎，
  Docker 部署，PWA 移动端适配
```

### 9.2 面试可讲的技术深度点

1. **为什么选 LangGraph 而不是手动编排？**
   - 状态管理清晰，支持人机协作（HITL）
   - 可以可视化 Agent 决策流程，方便调试

2. **RAG 的检索策略怎么设计的？**
   - 语义搜索（向量检索）+ 知识图谱关联检索 混合
   - 根据题目类型动态调整检索策略

3. **多 Agent 的上下文怎么管理？**
   - 使用 LangGraph 的 StateGraph 共享状态
   - 每个 Agent 只看到自己需要的状态字段（信息隔离）

4. **题目质量怎么保证？**
   - 结构化输出（Pydantic）保证格式正确
   - 题目生成后自动验证（答案是否在选项中）
   - 玩家反馈循环（答错率高 → 重新生成该题）
