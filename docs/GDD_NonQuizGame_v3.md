# 非答题型知识游戏设计文档 v3.1
# 赛博笔记 (CyberNote) 游戏模块 — 去答题化设计方案

> **设计理念**：学习不应该像考试，应该像探索、发现和创造。
> 本方案完全去除「答题」机制，用四种不同的游戏范式让玩家主动使用知识。
>
> **实现状态（2026-06-24）**：游戏盒子 (GameBox) 前端已实现。路由 `/game` → GameBox.vue（5张游戏卡片），`/game/arena` → GameArena.vue（现有知识防御战）。4个非答题方案以 BETA 标签呈现。

---

## 设计原则（去答题化宣言）

1. **知识是工具，不是考题** — 玩家用知识解决问题，而不是被考「你知道吗」
2. **主动建构 > 被动回忆** — 合成、连接、应用，而不是四选一
3. **发现驱动学习** — 「我找到了一个新连接！」比「我答对了」更让人上瘾
4. **知识即力量** — 知道得越多，游戏里越强大，而不是只是分数更高

## 方案一：概念炼金术（Concept Alchemy）
**类型**：合成解谜　**灵感**：Little Alchemy、2048　**开发成本**：★☆☆☆☆　**学习有效性**：★★★★★

### 核心玩法
玩家在「炼金工坊」里，将两张已掌握的 Wiki 概念卡片拖到一起，尝试合成新概念。

```
梯度下降 + 反向传播 = 神经网络训练（新 Wiki 节点自动生成）
决策树 + 熵         = ID3 算法
过拟合 + 正则化     = 岭回归
```

### 学习机制（无答题）

| 阶段 | 玩家在做什么 | 学习如何发生 |
|------|-------------|-------------|
| 选择阶段 | 思考「这两个概念有关系吗？」 | 主动检索知识关联（比被动答题有效 3 倍） |
| 合成判定 | LLM 评估两概念是否可合成 | 获得「为什么可以/不可以」的解释反馈 |
| 成功反馈 | 看到新概念 + LLM 生成的解释 | 获得新建构的理解，多巴胺奖励 |
| 失败提示 | LLM 建议「试试 X 和 Y」 | 获得探索方向，驱动去读更多 Wiki |

### 与 CyberNote 数据映射

| 游戏元素 | 数据来源 | 说明 |
|---------|---------|------|
| 炼金卡片 | `wikis` 表 | 每个 Wiki 节点 = 一张卡片 |
| 卡片能量消耗 | `knowledge_mastery.mastery_level` | 掌握度越高，使用成本越低 |
| 可合成关系 | `wiki_links` | 已有双向链接 = 高合成成功率 |
| 合成判定 | LLM 调用 | 用现有 Wiki 内容作为上下文判定 |
| 新概念生成 | LLM + 写回 `wikis` | 合成成功 → 自动创建新 Wiki 节点 |
| 合成历史 | 新建 `alchemy_history` 表 | 记录尝试历史，用于推荐薄弱知识 |

**Day 4**：能量系统 + 掌握度影响合成概率
**Day 5**：合成历史 + 推荐薄弱概念

## 方案二：Wiki 世界（Wiki World）
**类型**：探索冒险　**灵感**：Outer Wilds、Chants of Sennaar　**开发成本**：★★★☆☆　**学习有效性**：★★★★☆

### 核心玩法
你的 Wiki 知识图谱 = 游戏世界地图。每个 Wiki 节点 = 一个可探索的区域/房间。`wiki_links` = 区域之间的门/路径。

玩家在知识世界里自由行走，目标是通过探索理解整个知识体系。

### 世界生成规则

| 游戏元素 | 数据来源 | 生成规则 |
|---------|---------|---------|
| 区域/房间 | `wikis` 节点 | 每个节点 = 一个房间，内容渲染为环境叙事 |
| 门/通道 | `wiki_links` | 双向链接 = 两房间之间的门（双向通行） |
| 区域主题 | Wiki 分类/标签 | 同类概念聚在同一个「楼层」或「区域」 |
| 迷雾 | `knowledge_mastery` | 掌握度 = 0 的区域被迷雾覆盖 |
| 解锁条件 | 关联节点掌握度 | 要进入高级区域，需要先理解前置概念 |

### 学习机制（无答题）

玩家在 Wiki 世界里行走时：
- **读环境** = 读 Wiki 内容（渲染为「石碑」「卷轴」「NPC 对话」）
- **开门** = 理解两个概念的关联（wiki_links 的意义）
- **驱散迷雾** = 掌握新概念（mastery 提升）
- **发现彩蛋** = 找到隐藏的 wiki_links 连接

### 界面设计（文字描述）

```
┌──────┐
│  🏛️ 机器学习楼层           迷雾：62%           │
├──────┤
│       ┌────────┐                            │
│       │ 梯度下降 │←──门──┌──────────┐        │
│       └────────┘        │ 反向传播   │        │
│           ↑ 门           └──────────┘        │
│       ┌────────┐          ↓ 楼梯             │
│       │ 损失函数 │────门──→  [迷雾区域]       │
│       └────────┘                             │
│                                             │
│  📜 当前区域：梯度下降                       │
│  「通过调整参数让损失函数变小的方法...」      │
│  [阅读更多] [标记为已理解] [寻找连接]        │
└─────────────────────────────────────────────┘
```

### 开发计划（2-3 周）

**Week 1**：知识图谱 → 2D 地图生成算法（force-graph 布局或直接用现有 force-graph 组件）
**Week 2**：玩家移动 + 迷雾系统 + 区域解锁逻辑
**Week 3**：环境叙事渲染（Wiki 内容 → 游戏内可读对象）+ 成就系统

## 方案三：知识解谜（Knowledge Puzzle）
**类型**：环境解谜　**灵感**：The Witness、Blue Prince　**开发成本**：★★☆☆☆　**学习有效性**：★★★★★

### 核心玩法
游戏呈现一系列环境谜题，解开谜题需要理解 Wiki 里的知识。不是「答题」，而是「用知识操作游戏世界」。

例：
- 谜题：一个面板上有两个开关，分别标着「偏差大」和「方差大」
- 玩家需要理解「偏差 vs 方差」的区别才能操作正确
- 操作正确 → 门打开 → 进入下一关
- **没有「选择题」，只有「操作」**

### 谜题生成（LLM 驱动）

用 LLM 从 Wiki 内容自动生成谜题：

```
输入：Wiki 节点「偏差-方差权衡」的内容
输出：{
  "puzzle_type": "switch_panel",
  "elements": ["高偏差低方差", "低偏差高方差"],
  "correct_action": "根据问题类型选择对应开关",
  "hint_from_wiki": "偏差高 = 欠拟合，方差高 = 过拟合"
}
```

### 与答题的本质区别

| 答题 | 知识解谜 |
|------|---------|
| 题目：偏差和方差的区别是？ | 谜题：用偏差/方差知识操作开关开门 |
| 选 A/B/C/D | 直接操作游戏对象 |
| 答对 = 分数 +1 | 解开 = 进入新区域 + 叙事推进 |
| 是「考试」 | 是「用知识改变世界」 |

## 方案四：概念防御战（重设计，无答题）
**类型**：策略塔防　**开发成本**：★★★☆☆　**学习有效性**：★★★☆☆

### 与原来答题塔防的区别

| 原方案（答题版） | 新方案（无答题版） |
|----------------|------------------|
| 答题正确 = 塔攻击 | 塔自动攻击，不需要答题 |
| 答题错误 = 塔失效 | 塔的强度取决于概念掌握度 |
| 核心循环 = 答题 | 核心循环 = 策略部署 |

### 新核心玩法
- 敌人 = 误解/错误概念（「过拟合」「维度灾难」）
- 防御塔 = 已掌握的正确概念（「正则化」「降维」）
- 放置防御塔需要消耗「理解度」
- 塔的等级 = `knowledge_mastery.mastery_level`
- **没有答题，纯策略部署**

## 附录：为什么「不答题」更有效（认知科学基础）

1. **主动回忆效应**（Active Recall）：主动建构知识关联比被动识别答案深 3 倍
2. **生成效应**（Generation Effect）：自己「发现」的答案比被「告诉」的答案记忆深 2 倍
3. **情境学习**（Situated Learning）：在应用情境中学到的知识，迁移能力更强
4. **内在动机**（Intrinsic Motivation）：探索欲 > 考试欲，多巴胺驱动 > 分数驱动
