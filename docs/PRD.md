# 赛博笔记（CyberNote）产品需求文档

> 版本：v1.0 | 日期：2026-05-23 | 状态：草稿

---

## 一、产品定位

赛博笔记是一款面向个人用户的"笔记 + LLM 第二大脑"系统。用户在时间流中记录日常想法与原始素材，系统通过 Karpathy Wiki 编译范式，由大模型将碎片化笔记编译为结构化 Wiki 知识库，形成可持续积累、可对话查询的个人知识体系。

**核心理念（Karpathy Wiki 范式）**
- 原始素材只管「倒进来」，不要求实时整理
- LLM 定期或按需将原始笔记「编译」为结构化 Markdown Wiki
- 用户随时对话查询 Wiki，LLM 以已整理知识为上下文回答

## 三、核心功能模块

### 3.1 笔记时间流（Memo Flow）
- 支持 Markdown 格式输入
- 时间轴视图，按天/周/月浏览
- 支持标签（Tag）和类型（Type）标注
- 支持附件上传（图片、PDF）
- 支持置顶、归档、软删除
- 全文搜索

### 3.2 Wiki 知识库（Wiki Hub）
- 展示 LLM 编译后的结构化 Wiki 页面
- 按「领域 / 概念 / 实体 / 对比 / 来源」五类分类浏览
- 支持 Markdown 渲染，含 front matter 元数据展示
- 支持双向链接（`[[页面名]]` 语法解析与跳转）
- 知识图谱可视化（力导向图，`force-graph` 实现）
- 手动编辑 Wiki 页面（带版本历史）

### 3.3 LLM 编译引擎（Compile Engine）
- **触发方式**：手动触发 / 定时自动触发（可配置 Cron）
- **编译流程**：
  1. 读取未编译的笔记（`compiled=false`）
  2. 组装 Prompt（参照 TheSchema 规范）
  3. 调用 LLM API 生成 Wiki 页面
  4. 解析输出，写入 Wiki 数据库 + 本地 Markdown 文件
  5. 标记笔记为已编译
- **支持增量编译**：仅处理新增/修改的笔记
- **编译任务队列**：异步执行，前端可实时查看进度

### 3.4 AI 对话助手（Chat）
- 全局浮动对话入口
- 上下文：自动注入用户 Wiki 知识库摘要（非 RAG，直接 inject）
- 支持引用具体 Wiki 页面（`@页面名` 语法）
- 对话历史持久化，支持多会话管理
- 支持流式输出（SSE）

### 3.5 模型管理（Model Hub）
- 支持切换云端模型：
  - DeepSeek（`deepseek-chat` / `deepseek-reasoner`）
  - 阿里云百炼（`qwen-max` / `qwen-plus` / `qwen-turbo`）
- 支持本地 Ollama 模型（自动探测 `localhost:11434` 可用模型）
- 每个功能模块可独立配置使用的模型（如编译用 qwen-max，对话用 deepseek-chat）
- API Key 加密存储（AES-256）
- 模型可用性检测（连接测试）

### 3.6 功能模块扩展框架（Plugin Slots）
- 预留「模块插槽」架构，后期可在左侧导航新增功能模块（如求职模块、项目管理等）
- 每个扩展模块为独立 Vue 子应用或路由模块，不影响核心功能
- 后端预留 `pluginRouter` 扩展挂载点

## 四、非功能需求

### 4.1 性能
- 笔记列表首屏加载 ≤ 1s（分页，每页 20 条）
- Wiki 搜索响应 ≤ 300ms（FTS5 全文索引）
- LLM 编译为异步任务，不阻塞主线程

### 4.2 安全
- JWT 认证（本地单用户模式可关闭）
- API Key 本地 AES 加密，不明文落库
- 输入内容 XSS 过滤（后端 sanitize）

### 4.3 可迁移性
- 所有数据存储于本地 SQLite（开发/单机）或 PostgreSQL（生产/服务器）
- Wiki 同步输出为 Markdown 文件夹（`wiki/` 目录），可独立使用
- Docker Compose 一键部署，支持 ARM（手机/服务器）

### 4.4 移动端
- 前端响应式设计，适配 375px 以上屏幕
- PWA 支持（可添加到手机主屏幕）
- 触摸友好的交互设计

## 五、界面设计原则

- **极简风格**：只展示当前任务相关的内容，其余收折
- **双栏布局**：左侧固定导航（图标 + 文字），右侧内容区
- **暗色 / 亮色主题**：用户可切换
- **无多余动效**：过渡仅 fade-in 150ms，不用弹跳、粒子等装饰
- **字体**：正文 `Inter` / `Noto Sans SC`，代码块 `JetBrains Mono`

---

## 六、数据流

```
用户输入 → Memo Flow（时间流笔记）
                ↓ 触发编译
         Compile Engine（LLM 编译）
                ↓
         Wiki Hub（结构化知识）
                ↓ 注入上下文
         Chat 对话助手（LLM 回答）
```

## 七、验收标准

| 功能 | 验收条件 |
|------|---------|
| 笔记录入 | 输入 Markdown，保存成功，时间流显示正常 |
| LLM 编译 | 点击编译按钮，5 条测试笔记生成 5 个 Wiki 页面，格式符合 TheSchema |
| AI 对话 | 发送问题，30s 内流式返回基于 Wiki 上下文的回答 |
| 模型切换 | 在设置页切换 DeepSeek / Qwen / Ollama，下一次对话生效 |
| Wiki 浏览 | 双向链接可跳转，知识图谱可渲染 100 个节点不卡顿 |
| 移动端 | iPhone 14 / Android 竖屏核心功能可用 |
| Docker 部署 | `docker compose up` 后访问 8080 端口所有功能正常 |
